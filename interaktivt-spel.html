Interaktivt Rymdspel

Objektorienterat rymdspel med boss fights

Object-oriented space game with boss fights

I det här projektet ville vi skapa ett spel som kombinerar action, rymd och strategi – inspirerat av klassiska arkadspel som Galaxy Attack och Space Invaders, men med vår egen twist. Spelet utvecklades som en del av en kurs i objektorienterad programmering med fokus på att använda externa bibliotek.

Spelaren styr ett rymdskepp som måste undvika och skjuta ner fallande meteorer, och efter en viss poängsumma dyker ett stort monster upp som fungerar som en boss. Ju mer spelaren skjuter på monstret, desto fler poäng får man.
In this project, we wanted to create a game that combines action, space, and strategy – inspired by classic arcade games like Galaxy Attack and Space Invaders, but with our own twist. The game was developed as part of a course in object-oriented programming focusing on using external libraries.

The player controls a spaceship that must avoid and shoot down falling meteors, and after a certain score, a large monster appears that functions as a boss. The more the player shoots at the monster, the more points they earn.

Spelkoncept och uttryck

Game Concept and Expression

Vi skapade ett spel som balanserar retroinspiration med moderna element. Med inspiration från klassiska 2D-spel kombinerade vi den nostalgiska känslan med dynamiska visuella effekter, ljud och rörelser för att skapa en engagerande upplevelse.

We created a game that balances retro inspiration with modern elements. With inspiration from classic 2D games, we combined the nostalgic feeling with dynamic visual effects, sounds, and movements to create an engaging experience.

Det som gör vårt uttryck intressant är balansen mellan det retroinspirerade och det moderna. Vi har tagit inspiration från äldre 2D-spel men försökt skapa en mer dynamisk känsla. Begrepp som "reaktivitet", "spelmekanik" och "progression" beskriver väl vårt uttryck – varje del av spelet ska svara på spelarens handlingar och belöna skicklighet.

What makes our expression interesting is the balance between retro-inspired and modern. We've taken inspiration from older 2D games but tried to create a more dynamic feel. Concepts like "reactivity", "game mechanics", and "progression" well describe our expression – every part of the game should respond to the player's actions and reward skill.

Kärnelement i spelet:

Core elements of the game:

  • Spelstyrning med tangentbord eller mus
  • Progresiv svårighetsgrad med ökande meteorhastighet
  • Boss fight-system som aktiveras vid poänggräns
  • Omfattande poängsystem med kombos och multiplikatorer
  • Game control with keyboard or mouse
  • Progressive difficulty with increasing meteor speed
  • Boss fight system activated at score threshold
  • Comprehensive scoring system with combos and multipliers

Teknisk implementering

Technical Implementation

Projektet implementerades med objektorienterad programmering i JavaScript med hjälp av p5.play-biblioteket. Arbetet involverade skapandet av flera komplexa klasser och hantering av realtidsinteraktioner.

The project was implemented with object-oriented programming in JavaScript using the p5.play library. The work involved creating several complex classes and handling real-time interactions.

Tekniska utmaningar och lösningar:

Technical challenges and solutions:

  • Objektorienterad struktur: Vi började med att skapa de grundläggande klasserna för spelaren, projektiler, meteorer och bossen. Att arbeta objektorienterat hjälpte oss att hålla koden organiserad, men det tog tid att förstå hur instanser och metoder fungerade tillsammans i p5.play.
  • Kollisionsdetektering: En av de största utmaningarna var att få kollisioner och rörelser att fungera smidigt mellan objekten. Vi implementerade ett effektivt kollisionssystem som kontrollerar när projektiler träffar meteorer eller bossen.
  • Statemanagement: Implementering av olika spelstatusar (start, spel, paus, game over, boss fight) krävde noggrant tillståndshanteringssystem.
  • Object-oriented structure: We started by creating the basic classes for player, projectiles, meteors, and the boss. Working object-oriented helped us keep the code organized, but it took time to understand how instances and methods worked together in p5.play.
  • Collision detection: One of the biggest challenges was getting collisions and movements to work smoothly between objects. We implemented an efficient collision system that checks when projectiles hit meteors or the boss.
  • State management: Implementing different game states (start, play, pause, game over, boss fight) required a careful state management system.

Designkritik och förbättringar

Design Critique and Improvements

Under designkritiken fick vi värdefull feedback som hjälpte oss att identifiera områden för förbättring och framtida utveckling.

During design critique, we received valuable feedback that helped us identify areas for improvement and future development.

Mottagen feedback:

Received feedback:

  • Tydlig idé och fungerande mekanik: Spelet hade en tydlig vision och välfungerande grundläggande spelmekanik som uppskattades av testare.
  • Visuell förbättringspotential: Feedback pekade på att spelet kunde bli ännu mer visuellt levande med fler effekter och animationer.
  • Ljuddesign: Förslag på förbättrad ljuddesign för att förstärka spelupplevelsen.
  • Clear idea and working mechanics: The game had a clear vision and well-functioning basic game mechanics that were appreciated by testers.
  • Visual improvement potential: Feedback pointed out that the game could become even more visually alive with more effects and animations.
  • Sound design: Suggestions for improved sound design to enhance the gaming experience.

Implementerade och planerade förbättringar:

Implemented and planned improvements:

  • Explosionseffekter: Ett förslag var att lägga till fler visuella effekter, exempelvis explosioner när meteorer träffas eller när bossen tar skada. Vi hann inte implementera allt detta, men planerar att lägga till det i framtida versioner.
  • Partikeleffekter: Ytterligare förslag inkluderade partikeleffekter för projektiler och träffmarkeringar.
  • UI-förbättringar: Förbättringar av användargränssnittet för bättre spelarinformation och navigering.
  • Explosion effects: One suggestion was to add more visual effects, such as explosions when meteors are hit or when the boss takes damage. We didn't have time to implement all of this, but plan to add it in future versions.
  • Particle effects: Additional suggestions included particle effects for projectiles and hit markers.
  • UI improvements: Improvements to the user interface for better player information and navigation.

Resultat och lärdomar

Results and Learnings

Slutresultat: Vi utvecklade ett fullt fungerande interaktivt rymdspel med avancerade spelmekaniker inklusive boss fights, progressiv svårighetsgrad och omfattande poängsystem. Spelet demonstrerar effektiv användning av objektorienterad programmering och externa bibliotek.

Tekniska lärdomar: Projektet har gett oss en bättre förståelse för objektorienterad programmering och hur man bygger upp ett spel från grunden. Vi har lärt oss:

  • Hur man strukturerar komplexa applikationer med klasser och objekt
  • Användning av externa bibliotek (p5.play) för spelutveckling
  • Implementering av realtids kollisionsdetektering
  • Statemanagement i spelapplikationer
  • Optimerings tekniker för prestanda i spel

Design- och processlärdomar:

  • Vikten av iterativ design och kontinuerlig testning av spelmekanik
  • Hur ljud, grafik och spelmekanik tillsammans skapar en helhetsupplevelse
  • Balansering av svårighetsgrad för att hålla spelare engagerade
  • Värde av användarfeedback i spelutvecklingsprocessen

Framtida möjligheter: Projektet har lagt en solid grund för fortsatt spelutveckling. Potentiella nästa steg inkluderar:

  • Utöka spelet med fler banor och bossar
  • Implementera power-ups och specialförmågor
  • Skapa en online highscore-tabell
  • Porta spelet till mobilplattformar
  • Utveckla multiplayer-funktionalitet

Final result: We developed a fully functional interactive space game with advanced game mechanics including boss fights, progressive difficulty, and comprehensive scoring systems. The game demonstrates effective use of object-oriented programming and external libraries.

Technical learnings: The project has given us a better understanding of object-oriented programming and how to build a game from scratch. We have learned:

  • How to structure complex applications with classes and objects
  • Use of external libraries (p5.play) for game development
  • Implementation of real-time collision detection
  • State management in game applications
  • Optimization techniques for game performance

Design and process learnings:

  • The importance of iterative design and continuous testing of game mechanics
  • How sound, graphics, and game mechanics together create a holistic experience
  • Balancing difficulty to keep players engaged
  • Value of user feedback in the game development process

Future opportunities: The project has laid a solid foundation for continued game development. Potential next steps include:

  • Expanding the game with more levels and bosses
  • Implementing power-ups and special abilities
  • Creating an online highscore table
  • Porting the game to mobile platforms
  • Developing multiplayer functionality
